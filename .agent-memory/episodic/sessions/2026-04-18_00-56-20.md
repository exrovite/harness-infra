### What Was Built
5 deliverables implementing periodic evidence checkpoints with harness-injected verifier context.

**D1: create-evidence-checkpoint.sh** (NEW — ~150 lines)
- Builds checkpoint brief from must-do source files + summary + modified files
- Reads must-do index, extracts listed file paths, includes content (truncated 3000 chars each)
- Includes agent-provided paths from evidence-paths.json (iterative feedback loop)

**D2: pre-write-gate.sh** — Evidence checkpoint block
- Blocks source code writes when evidence-checkpoint.json has status:pending
- On PASS verdict: clears checkpoint + verdict + paths files, resets counter
- On FAIL verdict: shows specific per-phase findings, tells agent about evidence-paths.json
- No verdict: tells agent to spawn verifier sub-agent
- Exempt: .claude/state/, .claude/evidence/, Agent tool, all existing exemptions
- Also added .claude/evidence/ to contract gate exemptions

**D3: pre-bash-gate.sh** — Mirror checkpoint block
- Same logic as D2 for file-writing Bash commands

**D4: on-prompt-submit.sh** — Checkpoint injection
- When checkpoint pending: injects "[EVIDENCE CHECKPOINT] Writes blocked. Spawn a verifier..."
- 143 bytes (under 300-char limit)
- Sub-agents in same project see the injection via their own prompt-submit hook

**D5: post-write-check.sh** — Counter + trigger
- Increments checkpoint-counter.json on each write
- Triggers checkpoint at threshold (default 15, configurable via checkpoint-config.json)
- Triggers immediately on watcher step change (reuses pre-flight gate-counter for step)
- Resets counter after successful trigger

### Design Decisions
- Harness injects must-do SOURCE FILES into verifier brief (not just agent's summary)
- Builder cannot filter what verifier sees — harness controls the brief
- Iterative feedback loop: FAIL names specific phases, agent can provide file paths via evidence-paths.json
- Counter reuses pre-flight gate-counter.json for step tracking (no duplicate watcher reads)
- Generic: works for any project with a must-do folder, zero impact without one
