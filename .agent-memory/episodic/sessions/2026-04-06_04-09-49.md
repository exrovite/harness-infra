# Progress Notes — Pre-Flight Gate System

## Status: PLAN COMPLETE — Ready for NEGOTIATE

## What Was Designed
Two-system pipeline to prevent agent drift during multi-step tasks:

1. **Watcher + Cron (enhanced)** — Watcher slot becomes the ground truth anchor. Enhanced with SCOPE, OUT OF SCOPE, and MISTAKES TO AVOID sections. Cron stays at 3 minutes to catch inactivity.

2. **Pre-Flight Gate (new)** — PreToolUse hook on Write|Edit that blocks every write until agent passes a 4-question multiple-choice quiz derived from its own watcher slot. Response consumed after each write — must re-answer for every subsequent write.

3. **Distractor Pool** — Rotating library of ~100+ plausible wrong answers per category (tasks, steps, files, constraints). Random selection via `shuf -n 3`. No stored answer key — correct answers re-derived from watcher slot at validation time.

## Key Design Decisions
- MCQ over free-text: deterministic validation (exact A/B/C/D match), can't be bluffed with vague filler
- Response consumed on use: forces re-grounding on EVERY write, not just once
- No stored answer key: correct answers computed from watcher slot, agent can't cheat by reading the key
- Distractor rotation: agent can't memorize wrong answers across writes
- Pre-flight directory exempted: agent can write response.md without triggering the gate
- Cron stays at 3 min: catches "going quiet" which the gate can't (no writes = no gate fire)

## Origin
Generalized from the experiment-gate pattern (project-specific PreToolUse hook that blocked Write/Edit on experiment scripts until a pre-flight.md was filled with Ground Truth, Mistakes to Avoid, Protocol, Changes Planned). The insight: the watcher slot filled at task start IS the ground truth, and MCQ prevents bluffing.
